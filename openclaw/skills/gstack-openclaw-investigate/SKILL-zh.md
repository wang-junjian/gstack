# 系统性调试

## 铁律

**没有根本原因调查的修复。**

修复症状会创建打击地鼠调试。每个不处理根本原因的修复都会使下一个bug更难找到。找根本原因，然后修复它。

---

## 第1阶段：根本原因调查

在形成任何假设前收集背景。

1. **收集症状：** 读错误消息、堆栈跟踪和再生产步骤。如果用户没有提供足够的境，一次问一个问题。不要一次问五个问题。

2. **读代码：** 从症状向潜在原因追踪代码路径。搜索所有引用，读失败点周围的逻辑。

3. **检查最近的改变：**
   ```bash
   git log --oneline -20 -- <affected-files>
   ```
   这在工作之前吗？什么改变了？回归意味着根本原因在diff中。

4. **再生产：** 你能确定性地触发bug吗？如果不，在继续前收集更多证据。

5. **检查内存**关于同一地区的先前调试会话。同一文件中的反复出现的bugs是架构气味。

输出：**"根本原因假设：..."** ... 什么是错的和为什么的具体、可测试的声明。

---

## 第2阶段：模式分析

检查此bug是否匹配已知模式：

**竞态条件** ... 间歇性、时间相关。查看对共享状态的并发访问。

**Nil/null传播** ... NoMethodError、TypeError。缺失对可选值的保护。

**状态腐蚀** ... 不一致的数据、部分更新。检查事务、回调、钩子。

**集成失败** ... 超时、意外响应。外部API调用、服务边界。

**配置漂移** ... 在本地工作，在暂存/prod失败。环境变量、功能标志、数据库状态。

**陈旧缓存** ... 显示旧数据、缓存清除时修复。Redis、CDN、浏览器缓存。

也检查：
- 项目中的已知问题关于相关问题
- Git日志解决同一地区之前的修复。同一文件中的反复错误是架构气味，不是巧合。

**外部搜索：** 如果bug不匹配已知模式，在线搜索错误类型。**清理首先：** 剥离主机名、IPS、文件路径、SQL、客户数据。搜索错误类别，不是原始消息。

---

## 第3阶段：假设测试

在写任何修复前，验证你的假设。

1. **确认假设：** 在被怀疑的根本原因添加临时日志陈述、断言或调试输出。运行再生产。证据匹配吗？

2. **如果假设是错的：** 搜索错误（清理敏感数据首先）。返回第1阶段。收集更多证据。不要猜。

3. **3-罢工规则：** 如果3个假设失败，**停止**。告诉用户：

   "3个假设测试，无一匹配。这可能是架构问题而不是简单bug。"

   选项：
   - 继续调查一个新假设（描述它）
   - 升级给人工审查（需要知道系统的人）
   - 添加日志并等待（工具区域并下次抓住它）

**红旗** ... 如果你看到任何这些，减速：
- "现在快速修复" ... 没有"现在"。正确修复它或升级。
- 在追踪数据流前提议修复 ... 你在猜。
- 每个修复都在别处揭示新问题 ... 错误的层，不是错误的代码。

---

## 第4阶段：实现

一旦根本原因被确认：

1. **修复根本原因，不是症状。** 消除实际问题的最小改变。

2. **最小diff：** 最少文件触及，最少行改变。抵抗重构邻近代码的冲动。

3. **写回归测试**那：
   - **失败**不修复（证明测试是有意义的）
   - **通过**带修复（证明修复工作）

4. **运行完整的测试套件。** 无回归被允许。

5. **如果修复触及>5文件：** 在继续前标记到用户的爆炸半径。那大关于bug修复。

---

## 第5阶段：验证和报告

**新鲜验证：** 再生产原始bug场景并确认它是固定的。这不是可选的。

运行测试套件。

输出一个结构化调试报告：

**调试报告**
- **症状：** 用户观察的
- **根本原因：** 什么实际上是错的
- **修复：** 什么改变了，带文件参考
- **证据：** 测试输出、再生产显示修复工作
- **回归测试：** 新测试的位置
- **相关：** 同一地区的先前错误、架构笔记
- **状态：** 完成 | 完成_有关切 | 阻断

保存报告到 `memory/` 带今天的日期，所以未来会话可以参考它。

---

## 重要规则

- **3+失败的修复尝试：停止和提质架构。** 错误的架构，不是失败的假设。
- **永不应用你不能验证的修复。** 如果你不能再生产和确认，不运送它。
- **永不说"这应该修复它。"** 验证并证明它。运行测试。
- **如果修复涉及>5文件：** 在继续前标记到用户。
- **完成状态：**
  - 完成 ... 根本原因发现，修复应用，回归测试写，所有测试通过
  - 完成_有关切 ... 固定但不能完全验证（例如，间歇性bug，需要暂存）
  - 阻断 ... 根本原因不清楚在调查后，升级
